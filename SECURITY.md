# Security Policy

## 🔒 Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it through the **GitHub Private Vulnerability Reporting** feature enabled on the repository.

### If Private Vulnerability Reporting is Disabled or Unavailable:
1. Open a minimal public GitHub Issue titled: `Request for private security communication channel`.
2. Do **NOT** include any details of the vulnerability, exploit steps, API keys, credentials, or server URLs in that issue.
3. The maintainers will respond promptly to establish a secure, private communication channel with you.

---

## 🛑 Security Constraints & Safeguards

To prevent accidental disclosure or exploitation of systems:
* **Never post API keys, connection URLs, credentials, or exploit details in public issues or discussions.**
* If you need to share diagnostics, logs, or screenshots:
  * Redact your VPS server URL, domain, and IP address.
  * Completely redact your API keys, tokens, passwords, and private parameters.
* Public production servers should be accessed only via HTTPS. Plain HTTP is restricted to local hosts or trusted private/local networks where explicit consent has been given.
